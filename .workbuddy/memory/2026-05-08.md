### 核心 API
```cpp

Future<std::vector<dns::AddrInfo>> GetAddrInfo(
    const std::string& host, int port,
    int family = AF_UNSPEC, int type = 0,
    Protocol protocol = Protocol::kAny,
    EventLoop* loop = nullptr);

// 逆向 DNS
Future<dns::NameInfo> GetNameInfo(
    const std::string& address, int port,
    int flags = 0, EventLoop* loop = nullptr);

// 同步版本
std::vector<dns::AddrInfo> GetAddrInfoSync(host, port, ...);
dns::NameInfo GetNameInfoSync(address, port, ...);
```

### AddrInfo 结构
```cpp
struct AddrInfo {
  int family;        // AF_INET 或 AF_INET6
  int type;          // SOCK_STREAM 或 SOCK_DGRAM
  int protocol;      // IPPROTO_TCP/IPPROTO_UDP
  std::string canonical_name;  // 规范主机名
  std::string address;         // IP 地址字符串
  int port;                    // 端口号
};
```

## 完成：TimeoutAt

### 修改文件
- `include/asyncio/wait.h` - 添加 TimeoutAt 类

### TimeoutAt API
```cpp
template <typename T>
class TimeoutAt {
 public:
  // 构造函数：指定绝对截止时间
  TimeoutAt(std::chrono::steady_clock::time_point deadline, Task<T>& task);

  // 析构函数：取消定时器，如果超时已触发则调用 Uncancel()

  // 返回超时是否已触发
  bool Fired() const;
};
```

### 测试结果
- 3 个 TimeoutAt 测试全部通过
- FastOperationSucceeds: 快速操作在超时前完成
- SlowOperationTimesOut: 慢速操作被正确取消
- DeadlineInPastFiresImmediately: 过去的截止时间立即触发

### 实现说明
- 使用 EventLoop::CallAt() 在绝对时间点调度取消
- 与 TimeoutScope 相同的 RAII 模式
